                                                                                                                                            adeshpande3 Mod Shiv Gehlot • 2 months ago

                                                                                                                                            You can think of a feature map as being the output of convolving a filter over an input layer. One 5x5x3 filter applied to a 32x32x3 image, will give you a 28x28x1 feature map which is made up of activation values. If you apply 10 filters to that image, then you'll get an output volume of 28x28x10, which basically means 10 feature maps. What they're doing is they're keeping the activations in one feature map (only 1 28x28 section) and disregarding the other 9 activation maps from that layer.

                                                                                                                                                              So, from what I understand, they're trying to combat the issue that one filter is activating a lot (you can see the almost white filter in Figure 6a). So, for each filter in a particular layer, then calculate its RMS value (unsure of how exactly that gets calculated). I guess you can think of it as just a special kind of average of all the weight values. As for the radius part, I'm not exactly certain. My gut feeling is that if the RMS value we calculate is outside/greater than some threshold, then the weights will be adjusted such that it is in the threshold/radius.
